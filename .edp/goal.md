# Goal

Build a session continuity tool for AI-native development workflows that helps developers pick up where they left off across Claude sessions without re-explaining context. The tool should capture, store, and restore session state so each new Claude session starts with full context of what was happening, what decisions were made, and what's next.

## Success criteria
- [ ] Starting a new Claude session with a project auto-loads full context — no manual pasting or summarizing needed
- [ ] Why-choices (architecture, tradeoffs, rejected ideas) survive across sessions and are surfaced when relevant
- [ ] Every session ends with a written "next steps" block that becomes the opener of the following session
- [ ] Integrates into the existing Claude Code CLI workflow (hooks, CLAUDE.md, memory) — no new UI required

## Out of scope
- Automatic cloud sync — local filesystem only for now
- Multi-user / team sharing — single developer continuity only
