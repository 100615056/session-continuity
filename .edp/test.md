# Test

**Goal:** Build a session continuity tool for AI-native development workflows that helps developers pick up where they left off across Claude sessions without re-explaining context. The tool should capture, store, and restore session state so each new Claude session starts with full context of what was happening, what decisions were made, and what's next.

---

## Feature checklist

### Happy path
- [x] `sc init` creates `.claude/`, `session.md` placeholder, `CLAUDE.md` @import, Stop hook
- [x] `sc snapshot` writes a dated session entry with git state + narrative
- [x] `sc status` reads and displays current `session.md`
- [x] `sc clear` resets `session.md` to placeholder (with confirmation)
- [x] `sc rotate` manually triggers snapshot write
- [x] New Claude session auto-ingests context via `CLAUDE.md` @import (verified live)

### Edge cases
- [x] `sc init` is idempotent — running twice produces no duplicate @import or Stop hook
- [x] `sc init` preserves pre-existing hooks in `settings.json`
- [x] `sc init` prepends @import without clobbering existing `CLAUDE.md` content
- [x] Rolling window: after 4 snapshot runs, only the 3 newest sessions are kept
- [x] `sc snapshot` works in a directory with no git repo (graceful fallback)
- [x] No `.tmp` file left behind after atomic write

### Error / failure paths
- [x] `claude -p` unavailable → objective-only snapshot written, `[unavailable]` note included
- [x] `parseSessions` ignores `<!-- sc: placeholder -->` comment lines
- [x] `atomicWrite` never leaves a partial file on success

### Regressions
- [x] `trimToWords` correctly handles: under limit, at limit, over limit, empty string
- [x] `serializeSessions` / `parseSessions` round-trip is lossless for 1 and N sessions
